Componentes de Interfaz (UI)

Desglosando los átomos que componen la experiencia digital.

Botones (Interactividad)

Anatomía de un Botón

Los botones son los detonadores de acción. Deben parecer "clicables" (Affordance). Un buen sistema de botones define una jerarquía clara:

  • Primarios: Rellenos con color fuerte. Solo uno por pantalla idealmente (la acción principal).
  • Secundarios: Contornos (outline) o tonos suaves. Para acciones alternativas (ej. "Cancelar").
  • Terciarios: Texto solo (Ghost buttons). Para enlaces menos importantes.

Estados clave: Hover (al pasar el mouse), Active (al presionar), Focus (teclado) y Disabled (inactivo).

Cards (Contenedores)

Organización Modular

Las "Cards" agrupan información relacionada en un bloque coherente. Son el estándar moderno gracias al diseño responsivo, ya que se apilan fácilmente en móviles.

Deben tener bordes definidos o sombras para separarse del fondo ("elevación"). Todo su contenido debe estar relacionado a un solo tópico (ej. un producto, un artículo).

Tip de Diseño: Evita sobrecargar la card. Si hay demasiada información, usa un botón "Ver más" o un modal.

Inputs (Entrada de Datos)

Conversación con el Usuario

Los inputs permiten al usuario "hablar" con el sistema. La regla de oro es la prevención de errores: usa etiquetas (labels) claras siempre visibles.

Valida los datos en tiempo real (mientras escriben) en lugar de mostrar errores crípticos al final. Usa el tipo correcto de input (`email`, `tel`) para facilitar la escritura en móviles.
